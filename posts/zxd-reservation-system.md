## 项目概述

知行岛自习室是一个面向社区的自习空间，需要一套完整的线上预约 + 到店核销系统。我作为前端开发者，独立完成了从后端 API、管理后台到微信小程序的全部开发与部署。

**核心链路**：用户扫码进入小程序 → 选座预约 → 到店蓝牙开门 → 离座结算。

## 技术选型

| 层级 | 技术 | 选型理由 |
|------|------|----------|
| **小程序** | 微信原生 + 通通锁插件 | 零额外依赖，蓝牙开锁必须走厂商插件 |
| **后端** | Python FastAPI | 异步高并发，自动生成 OpenAPI 文档 |
| **数据库** | SQLite（WAL 模式） | 日活 < 20，完全够用，零运维 |
| **ORM** | SQLAlchemy 2.0 | 类型安全，异步支持 |
| **管理后台** | Vue3 + Vite + Element Plus | 快速出活，组件库齐全 |
| **部署** | Ubuntu + systemd + Cloudflare Tunnel | 一台旧电脑搞定，零成本域名 |
| **AI** | DeepSeek API | 便宜、中文好、直连不用代理 |

## 部署架构

```
Ubuntu 22.04 Server (GTX 1050Ti · 7GB RAM)
│
├── FastAPI (systemd 用户服务)
│   └── 监听 :8000 → api.islandspace.xyz
│
├── 管理后台 (Nginx 静态文件)
│   └── 监听 :8082 → admin.islandspace.xyz
│
└── Cloudflare Tunnel
    └── 4 个子域名，全走 HTTPS
```

关键设计决策：
- **不用 Docker**：服务器内存只有 7GB，Docker 开销太大，裸跑更稳定
- **不用 Nginx 反代 API**：Cloudflare Tunnel 直接映射端口，少一层代理
- **SQLite 而非 MySQL**：单机低并发场景，SQLite 在 WAL 模式下性能足够

## 踩坑记录

### 1. 蓝牙 SDK 集成

通通锁插件的微信小程序版本经历了多次变更：
- 插件版本从 `3.1.7` 降到 `3.1.4` 才稳定
- API 从 `startBleToLock` 改为 `controlLock`
- 手机蓝牙搜不到锁 → 不是代码问题，纯粹距离/信号问题

### 2. 图片加载

小程序里 `image` 组件在真机上需要走 `wx.downloadFile` 下载到本地，不能直接用 HTTPS URL。开发者工具里正常 ≠ 真机正常。

正确处理流程：
```js
// ❌ 直接返回相对路径（真机不显示）
if (isLocalPath(url)) return url

// ✅ 先补全域名，再判断是否需要下载
const fullUrl = resolveStaticUrl(url)
if (isLocalPath(fullUrl)) return fullUrl
if (canUseRemoteDirectly(fullUrl)) return fullUrl
return downloadFile(fullUrl)
```

### 3. 数据库列缺失

代码迭代时新增了模型字段，但忘记 ALTER TABLE。SQLite 不支持 `ALTER TABLE ADD COLUMN IF NOT EXISTS`，每次都得手动检查。

**教训**：用 Alembic 做数据库迁移，不要手动改表。

### 4. 微信登录

开发阶段一直用硬编码的 dev 账号（id=1，「刘少」），`WX_APP_SECRET` 没填。上线前必须配好 AppSecret，走 `wx.login() → code2session` 正式链路。
